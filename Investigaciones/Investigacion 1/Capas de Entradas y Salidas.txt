Descripción de las capas de un sistema de archivos
Las entradas y salidas (E/S) son fundamentales para la interacción de un sistema informático con el mundo exterior, permitiendo la comunicación con dispositivos como discos, teclados, pantallas y redes. Estas se gestionan a través de una estructura en capas en el sistema operativo, donde cada capa abstrae la complejidad subyacente y ofrece una interfaz uniforme. La capa de hardware de E/S está compuesta por buses, controladores de dispositivos y los propios dispositivos, donde los buses transmiten mensajes mediante patrones de voltaje, y las controladoras actúan como traductores entre el procesador y el hardware, ejecutando comandos de alto nivel mediante la lectura y escritura de bits en registros. 

La comunicación entre el procesador y las controladoras puede realizarse mediante E/S programada, donde la CPU envía comandos y datos a través de instrucciones especiales, o mediante acceso directo a memoria (DMA), donde la controladora transfiere datos directamente a la memoria sin intervención de la CPU. Para coordinar la transferencia de datos, la controladora utiliza registros como el de estado, control, entrada de datos y salida de datos. Además, el procesador puede interactuar con la controladora mediante sondeo, en el que la CPU verifica continuamente el estado de la controladora, o mediante interrupciones, que permiten a la controladora notificar a la CPU cuando está lista. El sistema operativo proporciona una interfaz de E/S a las aplicaciones, ocultando las diferencias entre los dispositivos a nivel de hardware.

Un sistema de archivos es un sistema compuesto de múltiples niveles o capas, donde cada nivel se basa en las funciones de los niveles inferiores para crear nuevas funciones. Aquí hay una descripción de cada capa, comenzando con la capa más baja:
Control de E/S: 
Esta capa, la más baja, consiste en controladores de dispositivos y rutinas de manejo de interrupciones que transfieren información entre la memoria principal y el sistema de disco. Un controlador de dispositivo actúa como un traductor que convierte comandos de alto nivel (como "extraer bloque 123") en instrucciones de bajo nivel específicas del hardware para la controladora de hardware.
Sistema básico de archivos: 
Esta capa envía comandos genéricos al controlador de dispositivo apropiado para leer y escribir bloques físicos en el disco, identificados por su dirección numérica.
Módulo de organización de archivos: 
Esta capa entiende la estructura de los archivos y sus bloques lógicos y físicos. Realiza la traducción de direcciones lógicas de bloque a direcciones físicas de bloque utilizando información sobre el tipo de asignación de archivos y la ubicación del archivo. Esta capa también incluye el gestor de espacio libre, que rastrea los bloques no asignados y los proporciona al módulo de organización de archivos cuando se solicitan.
Sistema lógico de archivos: 
Esta capa define la interfaz del usuario y del programador con el sistema de archivos, incluyendo la definición de un archivo y sus atributos, las operaciones permitidas en los archivos y la estructura del directorio.
